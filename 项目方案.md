# HR 管理系统 — 项目总方案

> 基于 Axure 原型「系统模板一」设计，采用前后端分离架构，Docker 容器化部署。

---

## 一、项目概述

### 1.1 项目背景

基于 Axure RP 原型（`系统框架.html` + `登录界面.html`），构建一套完整的 **HR 人力资源管理系统**，涵盖招聘管理、员工关系、考勤、绩效、薪酬、系统设置六大核心模块，以及 CRM 客户管理、办公协同等辅助模块。

### 1.2 原型来源

| 组件 | 文件 |
|------|------|
| 登录页 | `登录界面.html` |
| 系统框架 | `系统框架.html` |
| 入口重定向 | `start_c_1.html` → `start.html#c=1` |
| 设计风格 | 蓝绿配色 + 左侧二级菜单 + 顶部面包屑 + iframe 内容区 |

### 1.3 开发团队

- **前端**：1 人（Vue 3 + TypeScript）
- **后端**：1 人（Java Spring Boot 3）

## 六、Docker 部署方案

### 6.1 容器编排

```yaml
# docker-compose.yml
services:
  mysql:
    image: mysql:8.0
    container_name: hr-mysql
    environment:
      MYSQL_ROOT_PASSWORD: root123
      MYSQL_DATABASE: hr_system
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
      - ./docker/mysql/init:/docker-entrypoint-initdb.d

  redis:
    image: redis:7-alpine
    container_name: hr-redis
    ports:
      - "6379:6379"

  minio:
    image: minio/minio
    container_name: hr-minio
    command: server /data --console-address ":9001"
    environment:
      MINIO_ROOT_USER: admin
      MINIO_ROOT_PASSWORD: admin123456
    ports:
      - "9000:9000"
      - "9001:9001"

  backend:
    build: ./hr-backend
    container_name: hr-backend
    ports:
      - "8080:8080"
    depends_on:
      - mysql
      - redis
      - minio

  frontend:
    build: ./hr-frontend
    container_name: hr-frontend
    ports:
      - "80:80"
    depends_on:
      - backend

volumes:
  mysql_data:
```

### 6.2 部署命令

```bash
# 启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f backend

# 停止
docker-compose down
```

## 七、前后端协作规范

### 7.1 API 规范

```
统一响应格式：
{
  "code": 200,        // 业务状态码
  "message": "成功",
  "data": {}          // 响应数据
}

RESTful 风格：
GET    /api/employees          → 列表查询
GET    /api/employees/{id}     → 详情
POST   /api/employees          → 新增
PUT    /api/employees/{id}     → 修改
DELETE /api/employees/{id}     → 删除
```

### 7.2 协作流程

```
1. 后端先定义接口 → 更新 Swagger 文档
2. 前端根据 Swagger 文档定义 API 层 → 并行开发
3. 联调阶段前后端对接 → 修复问题
4. 提交前各自自测 → Code Review
```

### 7.3 Git 分支策略

```
main          # 生产分支
├── dev       # 开发分支
│   ├── feature/阶段一-基础设施
│   ├── feature/阶段二-系统管理
│   └── ...
```

### 7.4 代码规范

| 端 | 工具 |
|----|------|
| 前端 | ESLint + Prettier + Husky |
| 后端 | Checkstyle + Alibaba Java Coding Guidelines |
